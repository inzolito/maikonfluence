Project Description

The project aims to upgrade three key legacy OP Pro 2.8 customers—South32 Worsley, Glencore Mangoola, and Kinross Bald Mountain—to OP Pro 4. This ensures no loss of critical functionality, unifies the codebase to reduce maintenance, and protects $1M in annual recurring revenue. By migrating existing customizations into a configurable platform, we deliver faster, more stable upgrades and improve long-term customer satisfaction. Key Elements:

1.Migration of Custom Features: Embed site-specific functionality in OP Pro 4.x through configuration-based customizations, minimizing code forks and tech debt.

2.Feature Parity: Close all functionality gaps between OP Pro 2.8 and 4.x to ensure operators retain essential workflows upon upgrade.

3.Standardization & Tech Debt Reduction: Consolidate to a single codebase for easier maintenance, quality assurance, and more efficient development.

4.Sustainable Upgrade Path: Enable faster, reliable updates for all customers, supporting new enhancements without disruptive, site-specific rework.
