Created by GUEDES Lays, last modified on oct 02, 2025


HIP (Hexagon Innovation Process)  Milestones are set to improve efficiency and effectiveness of the project, along with providing a controlled "gated approach" between various key project delivery points. As such, there are pre-defined requirements for each of the milestones agreed with various stakeholders during project planning (game).

Once agreed and set, the requirements and goal of the milestones cannot vary and/or change as it will disrupt the underlying work to achieve the outcomes. A formal change request has to be presented to the Project Manager for assessing any changes with the milestones, that will require sign off from the SMT.
