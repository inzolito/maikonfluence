Drill Designs

Create a Drill Pattern Design Importer

  1. Open MineEnterrpise Miint Admin Web Console
  2. Goto 00 Administration > 02 Data Management Queries > 
  3. Create a New query by clicking on the + icon: Drill - Import Drill Pattern
  4. Setup Column mappings and default values in relation to the Hole Data Design template.csv
    1. {A} - column A

    2. {/Code} - reference field code

    3. _Expression(Now()) - default system time

    4. {_User.Code} - default user login

  5. Save the Query design

Local Coordinate System

The local co-ordinate system used on the importer templates must be configured in the system.

This is required for local coordinate conversion to WS084 compatible coordinates

  1. Goto Location Management > 00 Configuration > LOC Coordinate System
  2. Create a  location coordinate system using the site EPSG code (www.egsp.org) or get site to provide


Coordinate Conversion Configuration

Local coordinates used on Drill pattern designed are converted via a scripting process on the MineEnterprise server during import.

Coordinate Conversion Source Files
https://hexmet.sharepoint.com/:f:/s/MIN-SERVICES-MiPlan/ErMch-9YpRlEug5eSToLCOkB7XmL0Xzrk5__1fDoPdLwdQ?e=wJ8Zyt

Scripts must be created for each Local coordinate system.

  1. On the MineEnterprise Application Server, goto C:\MiPlan\Integrations\ConvertCoordinates\
  2. If a folder does not exist for your local coordinate system, copy an existing folder and rename to the local coordinate system using a consistent folder naming convention.
  3. Edit the ConvertCoordinates.exe.config in a text editor such as Notepad++
  4. Confirm that the connectionString value is correct for your ME environment including the database name
  5. Update the FromProjectionArguments value.  This is typically obtained from the Coordinate System API site eg. https://www.epsg.io)
    e.g. Arizona East NAD83 PROJ.4: +proj=tmerc +lat_0=31 +lon_0=-110.1666666666667 +k=0.9999 +x_0=213360 +y_0=0 +ellps=GRS80 +towgs84=0,0,0,0,0,0,0 +units=m +no_defs
  6. Update the @Key parameter to your Coordinate description e.g. LastRun-HolPos-NAD83_Arizona_to_WGS84
  7. Update the @CoodSysFromId parameter to your local coordinate system e.g. CoordinateSystem = 'NAD83
  8. Update the second @Key parameter to the Local Coordinate description e.g. LastRun-HolPos-NAD83_Arizona_to_WGS84

Coordinate Conversion Scheduled Job

The Coordinate Conversion scripts are executed by scheduled job every 2 minutes on the MineEnterprise SQL Server.

  1. Connect to the MineEnterprise database server
  2. Open SQL Server Management Studio
  3. Open SQL Server Agent > Jobs and edit the MiiNT_Coordinate_Conversion_HolePositions job
  4. Add a new step to execute the batch file for the new Local Coordinate conversion script created in the previous section.

Testing the Coordinate Conversion

(NB: This process is managed by APAC services team not PD team)

If a Drill pattern design has been imported test the local coordinate conversion

  1. Run the batch file within the Local Coordinate system folder e.g. c:\MiPlan\Integrations\ConvertCoordinates\HolePositions_NAD83_Arizona_to_WGS84\ConvertCordinates.bat
  2. Confirm that records are successfully processed.
  3. Goto 05 Drill Production > Pattern - 04 Hole Positions >
  4. Confirm that records exist with WGS84 coordinates

If issues are experienced please contact MineEnterprise Support

Import Drill Pattern

  1. Goto Drill Production > 01 Import Patterns > Drill - Import Dril Pattern
  2. Import csv file
  3. Drill pattern will be displayed in preview mode
  4. Click on Save to commit to database

Manage Drill Pattern

  1. Goto Drill Production > 02 Manage Patterns > 
  2. Options available
    1. Pattern - 01 Pattern Information
    2. Pattern - 02 Hole Design
    3. Pattern - 03 Hole Intervals (only if customer use designs e.g. hardness profiles)
    4. Pattern - 04 Hole Positions


NOTE: Local coordinates must be converted to WGS84 coordinates (Lat, Log, Altitude) to work on the Foundation Apps.

Refer to Coordinate Conversion Section

Downloading Drill Pattern Designs on the Drill App

  1. Open and sign into the Drill App
  2. Select the side menu (1)
  3. Select the Download menu (2)
  4. Select the Drill Pattern Designs to download.  Click Download.(3)
  5. Select a Drill Pattern to use from the side menu. (4)
  6. The Drill Pattern will be loaded.  Click on the World icon if your require to navigate to the Shot on the map. (5)
  7. Select a Hole to display the Design values and start entering actuals

Blast Designs

Create a Blast Design Importer

  1. Goto Administration > 02 Data Management Queries > Queries
  2. Create New
  3. Name: Blast - Import Blast Deck Design
  4. Category: Patterns-ImportPatterns
  5. Table Code - Blast Deck Designs Import Flat - Bulk Cleardown (for flat csv files)
  6. Complete column mapping

Import Blast Design

  1. Goto Drill Production > 01 Import Patterns > Blast - Import Blast Deck Design
  2. Browse to the Blast Deck Design csv file and import
  3. Click on Save to commit the Blast Deck Design to the database

Manage Blast Design

  1. Goto 05 Drill Production > 012Manage Patterns > Pattern - 06 Blast Deck Designs
  2. Search for Blast Code

Configure Blast Designs Query on Blast App

  1. Goto 00 Administration > 03 Report Queries > MiBlast - Loading Shots
  2. Configure query so that Active and Visible Blast Designs are displayed
  3. Query XML

<?xml version="1.0" encoding="utf-16"?>
<QueryCommandItem Service="" Product="MiiNT" Table="BlaDecDes" Qmode="Summary" debug="false" resolve="true">
<Filters top="0" topPerc="false" distinct="false" xmlns="Query">
<FilterItem code="/Location/Vis" op="Between" name="LocIsVis" type="None" xmlns="Query.Filter">
<Filter val="Y" />
<Filter val="Y" />
<attr logic="None" bro="0" brc="0" ignorCase="false" calc="" descr="" options="" behav="None" having="None" />
</FilterItem>
<FilterItem code="/Location/Active" op="Equal" name="LocAct" type="None" xmlns="Query.Filter">
<Filter val="Y" />
<attr logic="None" bro="0" brc="0" ignorCase="false" calc="" descr="" options="" behav="None" having="None" />
</FilterItem>
<FilterItem code="/Quantity" op="Greater" name="Quantity" type="None" xmlns="Query.Filter">
<Filter val="0" />
<attr logic="None" bro="0" brc="0" ignorCase="false" calc="" descr="" options="" behav="None" having="None" />
</FilterItem>
</Filters>
<Outputs xmlns="Query">
<OutputItem code="/LocCod" dataType="None" func="None" showTotal="Auto" hidden="false" sort="0" name="LocCod" xmlns="Query.Output">
<attr cap="" hdr="" scale="0" nulls="false" filterLink="" calc="" format="" other="" group="" />
</OutputItem>
</Outputs>
</QueryCommandItem>

Make Blast Design Visible

  1. Goto 05 Drill Production > 02 Manage Patterns > Pattern Visibility
  2. Select the required designs to active

Downloading Blast Designs on the Blast App

  1. Open and sign into the Blast App
  2. Select the side menu (1)
  3. Select the Download menu (2)
  4. Select the Blast Designs to download.  Click Download.(3)
  5. If multiple designs are downloaded a message will be display requesting to select a design to use. (4)
  6. Select a Blast Design from the side menu. (5)
  7. The Blast Design will be loaded.  Click on the World icon if your require to navigate to the Shot on the map. (6)
  8. Select a Hole to display the Design values and start entering actuals
