Setting up an Android Dev Environment

  • Install OpenJDK 8.
    • Builds can be found here https://adoptopenjdk.net/. Install the relevant builds for the operating system.
    • If installing on windows, the JDK binary path will need to be set in the JAVA_HOME environment variable.
  • Install Android Studio
    • https://developer.android.com/studio
    • Using the SDK Manager, install SDK for at least Android 7, 8, 9,10 and 11.
    • use the SDK Manager to install Android SDK Tools, Platform-Tools and SDK Build Tools and the Android NDK tools
    • If installing on windows, the SDK path will needs to be set in ANDROID_HOME environment variables. the SDK\platform-tools and tools needs to be set in the PATH variables
    • If installing on windows, the NDK path will needs to be set in ANDROID_NDK_HOME environment variables. Considering the compatibility issue,  a legacy NDK version (ie 20.0.5594570) will needs to be downloaded using SDK Manager. 
    • Optionally ensure that Android Emulator is also installed.



Setting up Node Environment

Foundation apps are essentially Node Application that use Cordova to communicate and work in an Android environment. 

  • Install Node
    • Install the LTS from https://nodejs.org/en/. Which at the time of writing this doc is 14.17.1.
    • Optionally you could install NVM - https://github.com/nvm-sh/nvm.
      • This allows an easy mechanism to switch different versions of Node.


Install Mine Enterprise

Mine Enterprise is the back-end for the Foundation Apps.  There is already an installation guide  for setting up MineEnterprise.

  • The apps require some basic configurations to already exist within MineEnterprise. 
    • For sake of convenience , use one of the existing backups from the already existing customers.
    • The list of customers and which apps they use is maintained by the services team here.
      • If a more recent backup is required, or the existing backups don't work contact DUANE Chrisor ALVES Sergio


Setting up Cordova Environment

Starting version 4 all foundation apps list Cordova as a local dependency. However, to work with any previous versions of the apps Cordova will need to be installed globally. 

Recommended version for Cordova is 7.1.0. To install Cordova globally.

> npm i -g cordova@7.1.0


Setting up the Foundation Apps

Foundation Apps use git submodules for managing the Code base. When cloning the repository , ensure the submodules are also cloned.


> git clone --recurse-submodules git@bitbucket.org:miplandev/mihaul.git

Or Alternatively, in case you forget to clone the submodules

> git clone git@bitbucket.org:miplandev/mihaul.git
> cd mihaul
> git submodule update --init --recursive


Running the Foundation Apps in the Dev environment

Connect either one of the Android Tablets via USB to the machine or start an Android Emulator using Android Studio.

You can test this by using adb. Once all the android SDK is installed, you should have access to it using the command line


After cloning the repository, cd in to the directory and then install all the dependencies

> cd mihaul
> npm i

Once all the dependencies are installed you should be able to deploy the apps to either the emulator or the connected Tablet

> npm run android
