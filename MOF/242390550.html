Overview

All the Foundation Apps use git submodules to manage the code base. In general all apps comprise of two main submodules , i.e MiApp and MiTime. Each of these modules serve as a library and provide a mechanism to consume functionality that is shared across all the apps.

  • MiApp: Core of all the foundation apps
    • Provides common views and JQuery widgets used across the apps.
    • Provides utilities and services that allow the apps to variety of stuff, for example , make network requests, persist data to a local Sqlite storage, authenticate etc.
  • MiTime: Is an app by itself and also serves as a library to other Foundation Apps.
    • Provides UI Widgets , utilities and services that aid in Fleet Management.

All the views in the Apps follow an MVVM pattern and are built using  KendoUI. To build the application for an android device, Cordova is used. Latest versions of the apps use Cordova 9, which allows the apps to work in Android 7,8,9 and 10.



Initialisation

When one of the apps is installed for the first time on any tablet, it defaults to a settings view, which allows the administrator to enter key information required by the app to do its first initialisation successfully.

It requires two key pieces of information, namely the URL pointing to the location where MineEnterprise has been installed and a set of credentials it can use to login successfully. Once connected successfully it downloads three types of configuration.

  • Module Configs: These are configuration, that can be used to enable/disable certain functionality for the apps.
    • Each configuration is scoped. 
  • WritebackQueries: These queries provide enough metadata to the apps to enable it dynamically build UI's for collecting data.
  • Report Queries: These provide data to the apps to power features such as autocomplete, lookups etc.

Once it has done this initial download of the data, the apps persist it locally to a Sqlite database. This enables the apps to boot up offline , without a network connection
