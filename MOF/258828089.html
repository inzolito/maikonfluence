Product: MineOperate Foundation : How to: Upload data from Tablet to Server via Fiddler

Created by GEORGE Nathan, last modified by Usuario desconocido (babd) on mar 04, 2024


The instructions below will help to solve the issue when data is stuck on tablets and cannot be uploaded to the server. There are a few reasons why data can get stuck, see below.

Initial Troubleshooting

Data can get stuck because of a field or certain mandatory fields that have not been entered or entered incorrectly (different format), in most cases an error message will be thrown showing the fields that need to be fixed/updated with the correct values.

Users will need to go back to the data, review/update those values and then try re-uploading the data.

Also, sometimes it is possible that the tablet may be out of WIFI range, in that case users will need to get closer to a WIFI point or connect a network cable before uploading the data.

Another quick way that can sometimes help is to ask users to close the app and restart the tablet. If those tips cannot hep resolve the issues then data may need to be uploaded via Fiddler, see steps below.


Fiddler is a debugging proxy server tool used to log, inspect, and alter HTTP and HTTPS traffic between a computer and a web server or servers.

You will need to run Fiddler on the Server, but first you can download it from here .


Summary of how to uploading Tablet Data via Fiddler 

Step 1: Configure Fiddler first to run on port like 8888. (Tools → Options  Click on the Connections tab)

Step 2: On the client's tablet go to settings and change the tablet to send data to port 8888. 

Step 3: Close the tablet App and restart it then click to re-upload data.

Step 4: On Fiddler, capture the network traffic and save it. Or get the one that matter (is erroring)  and put it in reply mode. (Reply it to check that this works)

Step 5: Find out what is wrong with the network packet (if it looks like it is missing a Shift Name, or someone renamed the Shift??)

Step 6: Edit the data packet in a text editor

Step 7: Use fiddler to generate a new Session Id

Step 8: Replay the original network request, but manually put in the new Session Id and manually modify the packet of data so that it correct the problem from above.

Step 9: The data will be posted to the server

Step 10:  You can then wipe the data from the tablet as data has already been uploaded.

Step 11: Configure the tablet back to 8080 port and re-sync the tablet
