Product: MineOperate Foundation : Database

Created by Usuario desconocido (wchan), last modified on jun 09, 2022





Blend

  • Blend will be created with an excel workbook
  • Blend can be given a code (i.e blend 46)
  • Blend can be re-used (not unique)
  • Blend can be given a description
  • Has a start date and end date
  • Has active (Not visible on miblend)

(Contains the design information)

Recipe

  • Is assigned to Blend
  • many Recipe to one relationship to Blend
  • Source location is the design location of the recipe (i.e Stockpile type )
  • Destination location is the design location of the recipe (i.e ROM type)
  • Mass is the amount of movement required to complete the recipe
  • The order determines what is next


(The purpose of RecipeLoading is so that the Recipe_Id key do not have to live on Loading. This will make Loading generic for other purposes not specific to MiBlend)

RecipeLoading

  • Maintains the relationship between Recipe and Loading
  • The multiplicity intent is one Recipe can have many Loading
  • Multiple RecipeLoading can have one BlendSummary


(Purpose of BlendSummary is to keep track of the loadings done against a blend recipe iteration)

BlendSummary

  • Summary for Loading
  • Original% fields are used by the server to find the summary record given by the client


(Contains the actual information based on design)

Loading

  • Nothing worth mentioning on this table from a database point of view


For database schema

  • For fields that are nvarchar type
    • If they are code/description 60 characters
    • If they are comments, max.
  • Where not mentioned nullable, it is required
