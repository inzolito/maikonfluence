Fresh installation on tablet

NOTE: Reset and uninstall any already existing versions of Dig on the tablet before performing this installation!


  1. Obtain the latest Dig 4.0.0 APK
  2. Move the APK to the tablet (via a USB connection)
  3. Navigate to the APK using the tablet's file explorer
  4. Install Dig on the tablet using the APK
Dig installs successfully

Installation over existing version

NOTE: It might not be necessary to obtain and/or install an older version of Dig 4.0.0 if there is already an older version of Dig 4.0.0 installed on the tablet.


  1. Obtain both an older version of Dig 4.0.0 and the latest version of Dig 4.0.0
  2. Move both APKs to the tablet (via a USB connection)
  3. Navigate to the APKs using the tablet's file explorer
  4. Install the older version of Dig 4.0.0 on the tablet
  5. Start Dig, perform the first-time setup and perform a sync
  6. Install the latest version of Dig 4.0.0 on the tablet
The newer version of Dig installs successfully and retains all setup and sync data from the older version of Dig

Core Function (Digging Locations)

After logging-in, the user is able to download digging locations from the "Download" tab found in the left-hand sidebar.

These digging locations change the location and map found on the digging activity maps.

  1. Login
  2. Open the left-hand sidebar and select a digging activity
  3. Observe the location and map of the digging activity
  4. Open the left-hand sidebar and select the "Download" tab
  5. Select a digging location
  6. Tap the "Download" button
  7. Return to the previously selected digging activity and observe the location and map
  • The location changes to the digging location's location
  • The map changes to the digging location's map

Usuario desconocido (dask)


Core Function (Digging Locations)

Select a reason code from the left-hand sidebar.

The downloaded digging locations will select-able digging locations to the map, which the map will automatically center to when accessing the map (as long as the map isn't setup to center on the tablet's GPS location).

Tapping one of these digging locations will change its colour to indicate that it is selected and fill out the Pit, Strip and Block fields in the bottom-left locations box.

  1. Login
  2. Download some digging locations
  3. Open the left-hand sidebar and select a reason code
  4. Tap a few different digging locations
  • The downloaded digging locations are visible on the map
  • The map centers itself on the digging locations (if not set to center on the tablet's GPS position)
  • Tapping a digging location sets the Pit, Strip and Block fields in the bottom-left locations box

Usuario desconocido (dask)


Core Function (Reason Codes)

Select a reason code from the left-hand sidebar.

The equipment state buttons can be found in the footer, and the default state is "Ready".

Changing the equipment state increases the number of time allocation status items on the upload screen.

  1. Login
  2. Tap the "Ok" button on the prompt that appears
  3. Select a reason code
  4. Make some state changes
  5. Check the upload screen for the number of time allocation status items
  6. Upload the state changes (if auto upload hasn't triggered)
  7. Check the StagingTimeAllocations and EquipmentStates tables
  • Default equipment state is "Ready"
  • Changing equipment states increases the number of time allocation status items on the upload screen
  • All state changes are visible in both the StagingTimeAllocations and EquipmentStates tables
  • All state changes are in order in the EquipmentStates table

Core Function (Reason Codes)

Select a reason code from the left-hand sidebar.

The activity wheel has equipment activities and names for the activities, and the default activity is none. The number and names of these activities change depending on the currently selected reason code.

Changing the equipment activity increases the number of time allocation activity items on the upload screen.

  1. Login
  2. Tap the "Ok" button on the prompt that appears
  3. Select a reason code
  4. Take note of the number of activities and activity names on the activity wheel
  5. Make some activity changes
  6. Check the upload screen for the number of time allocation activity items
  7. Upload the state changes (if auto upload hasn't triggered)
  8. Check the StagingTimeAllocations and EquipmentActivities tables
  • Default activity is none
  • Each reason code has a different number of activities and has different activities on the activity wheel
  • Changing equipment activities increases the number of time allocation activity items on the upload screen
  • All activity changes are visible in both the StagingTimeAllocations and EquipmentActivities tables
  • All activity changes are in order in the EquipmentActivities table

Core Function (Reason Codes)

Select a reason code from the left-hand sidebar.

The map screen has a location for the user to enter, which is attached to all performed activities that are uploaded.

The field is user-determined and suggests locations that best match the user's entered text.

  1. Login
  2. Tap the "Ok" button on the prompt that appears
  3. Select a reason code
  4. Enter a location
  • A suggestion box should appear when typing text into the box that matches the names of existing and active locations
  • All equipment activity changes have the specified location's ID attached after being uploaded
  • Location IDs match a location in the LocationCode table
