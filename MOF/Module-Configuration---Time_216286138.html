MiTimeTrackEquipmentGPSInUserSessionOnlyTrueLogout

True - Shows the last known GPS position of the Equipment after the Operator logs out

False - Shows the location of the tablet, even when the Operators are logged out and the tablet is outside the Equipment


MiTimeAllowEquipmentHoursSkipTruePre-start

True - Allows the Operator to login to the Time app without entering start engine hours

False - The Operator goes to the Ready screen after login without entering engine hours


MiTimeAllowPreStartSkipTruePre-start

True - Allows the Operator to login to the Time app without entering the optional component condition checklist

False - The Operator is required to swipe-right on passed components, and swipe-left on damaged or missing components in the Pre-start checklist


MiTimeEnablePreStartTruePre-start

True - Activates the Pre-start checklist items from MineEnterprise 2.0 > Maintenance > Configuration > Pre-start Questions

False - The Operator goes to the Ready screen after login


MiTimeEndEquipmentHoursRequiredFalsePre-start

True - The Operator cannot logoff without entering engine hours greater than the start engine hours.

False - The Operator logs off with no further data entry.

Exception - If the Operator entered optional start engine hours, the Operator will not be able to logout without entering end engine hours.


MiTimeEnableLockScreenWhenMovingTrueScreen Lock

True - Prevents the Operator from using the Time app and displays a message "Equipment is moving" when the tablet detects movement over the Value set in LockScreenWhenMovingGreaterThanKmph

False - Ignores movement detected by the tablet. The Operator can use the tablet while moving.

Screen lock requires MineEnterprise 3.4.0, (3 Jun 2019) - minimum version

Time 3.0.0 / ME 3.4.0

2019 Jun

MiTimeHideLockScreenSpeedDecayKmph2Screen Lock

Set to a number between 2 to 6

  • To address poor GPS coverage
  • To unlock the Haul app when no GPS signal is detected

The Speed Decay ratio is used to estimate the amount of time needed to stop equipment when no GPS signal is available. The speed decay value is inverse

  • Lower decay value - longer time to unlock
  • Higher decay value - shorter time to unlock

After unlocking, the Haul app remains unlocked until a GPS signal can be used to calculate the movement of the equipment.

Time 3.0.0 / ME 3.4.0

2019 Jun

MiTimeLockScreenOnHideIgnoreShowRequestSeconds5Screen Lock

Add or remove 2 to 5 seconds

  • To address poor GPS coverage
  • Use to keep the screen unlocked after a full stop has been detected

The lock screen setting is the number of seconds the screen stays unlocked before sampling GPS data for movement.

Increase this number by 2-5 seconds if the Haul app is looping through locking and unlocking when the equipment is at a stop or in areas with poor GPS coverage.

MiTimeEnableLocationGeofences
Site MapLeave blank. Do not use. Use EnableGeofencesLocation instead
MiTimeGeofenceGpsMaxAccuracyValue10Site Map

Number of meters from GPS coordinates to identify the Equipment is inside a Location Boundary.

Requires EnableGeofencesLocation to be True, a GPS signal, and the GPS coordinates for polygons to be entered into Locations > Configuration > All Coordinates

Type a larger number for to compensate for poor GPS signal. Type a smaller number for strong GPS and smaller Location Boundaries.

MiTimeStartSessionReasonCodes
Time Allocation

Usually left blank in Time. This is the first State with Reason the tablet records when an Operator logs in.

Example of a Value:

Ready:Production


MiTimeStateWriteback41:writebackTime AllocationThe format and method to send the State (such as Ready, Standby, Delay, Down) from Time to MineEnterprise

Enter Boolean Values as Title-case, case-sensitive True or False. (A few of the values require the exact match.) 

Report: A number :report sends filtered and sorted data from MineEnterprise to be displayed on the Operator's tablet 

Writeback: A number :writeback sends data from the Operator's tablet to MineEnterprise

The numbers shown in this table are an example and will change based on each Client's installation.

If a Value is blank, the matching Key in MiApp controls the Vaue.
