Adding a DrillShot:

a Drill shot comes from the DrillShotsQuery. The recomended query for this selects from locations where the type is 'SHOT' where there are holes, are active, are visible, status != Blasted || Depleted, is WGS84 ( This is due to the tablets not converting to wgs84 on the fly )

Please note that Fields that are used in comparisons from a joined table seemingly cannot be null. for example the /location/status

Here is some SQL to setup a test drill shot with the minumum requirements for the default query:

Insert into Location (Code, Description, Active, CreatedDateTime, ModifiedDateTime, CreatedBy, ModifiedBy, LocationCategory_Id, LocationType_Id, Visible, [Status])

VALUES ( 'SHOT_01', 'SHOT_01', 1, GETDATE(), GETDATE(), 'Hexagon', 'MiPlan', 5, 6, 1, 'READY');

Insert into HoleParameters (HoleNumber, CreatedBy, ModifiedBy, CreatedDateTime, ModifiedDateTime, Location_Id, HoleStatus_Id) VALUES(1, 'Hexagon', 'MiPlan', GETDATE(), GETDATE(), 514, 1);

INSERT INTO HolePositions (DesignCollarX, DesignCollarY, DesignCollarZ, CreatedBy, ModifiedBy, CreatedDateTime, ModifiedDateTime, HoleParameters_Id, CoordinateSystem_Id) VALUES (

0, 0, 0, 'Hexagon', 'MiPlan', GETDATE(), GETDATE(), 2, 2);


Opening the map within the app, for the drill shot to be opened in the map view on the tablet, it will need to have atleast one valid hole. This is decided by the Drill Query. The default query requires that the hole parameter needs to have designCollarX and designCollarY that are not equal to 0. Aswell as a connected drill production record that has a drilled depth greater then 0. The following is example sql to get a working record for demonstration and testing.

Update HolePositions SET DesignCollarX = 1, DesignCollarY = 1 WHERE Id = 2;

INSERT INTO DrillProduction ( DrilledDepth, CreatedBy, ModifiedBy, CreatedDateTime, ModifiedDateTime, Equipment_Id, Period_Id, HoleParameters_Id, Date ) VALUES (1, 'Hexagon', 'MiPlan', GETDATE(), GETDATE(), 97, 1, 2, GETDATE());


The map at the moment requires having two drill points with in it other wise it does get max bounds on a single point creating a zoom level of infinitiy causing issues. So create 2 hole paramaters that are each linked to hole positions aswell as your location that you have setup.
