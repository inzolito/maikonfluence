MineEnterprise Haul Mass for MineOperate Foundation

The following applies to clients using MineEnterprise especially with MineOperate Foundation apps, for the Modules and Analytics:

  • Material Movement
  • Reconciliation
  • (Material) Inventory
  • Processing
  • Production

Issue - Error Message 

An error message is displayed in the MineEnterprise 2.0 Configuration of material Source and Destination Dashboards in MineEnterprise.

Error details Query metadata exception: None is not a supported Field Category setting.

Conditions

Applies to:

  1. MineEnterprise 2020.05 (3.6.1)
  2. MiiNT_990_Reporting.meta
  3. MineOperate Foundation apps with payload and mass data
  4. Sharperlight Query Builder configuration of
  • HaulMass, Material Movement data in Dashboards
  • May be required when creating dashboards with Time Allocation, Destination data

May not be compatible with OP Fleet Analytics (OP Pro, Jigsaw, and third-party integrations). 

Solution

In the MineEnteprise 2.0 Config, go to Standard Configuration > General > Lookup Values

2. Add two rows (go to the top-right, click Add New)

Table Code

Field Code

Value

Active

RLookups

PerHourReport

Accumulated

Yes

RLookups

PerHourReport

Sub total

Yes

  • Click Save

3. In Sharperlight > Query Builder, bot to the \Haul\Mass table and HaulMass output field

4. Select the General tab

  • 5. Verify the Show Null Values is checked

6. Complete the Query in Sharperlight

  • Export the XML from Sharperlight Query Builder
  • Place the XML into the MineEnterprise Dashboard Query
