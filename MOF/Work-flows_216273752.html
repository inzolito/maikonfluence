Product: MineOperate Foundation : Work flows

Created by Usuario desconocido (dask), last modified on ene 16, 2020

Login WorkFlow



  • Operator: Populated using the MiDig specific PersonCategoryFilter module config
  • Equipment: Populated using the MiDig EquipmentTypeFilter module Config
  • Shift: Day/Night Shift
  • All Three fields are required



PreStarts

Prestarts are configurable from Silverlight. 

  • EnablePreStart module config can be used to disable them. 
  • AllowPreStartSkip can be used to make them mandatory

Currently, Dig does not have any prestarts

Equipment Engine Hours 

Configured from AllowEquipmentHoursSkip module config. Is a True/False value

Configured from the EquipmentHoursWriteBack Query in Silverlight


Core WorkFlow

  • For the selected equipment type, if there are multiple Time Allocation Codes available, the user is prompted to make a selection.
    • If there is only one, the app defaults to the Ready state for that Allocation Code.


  • StartSessionReasonCode (miDig) is used to configure the default time Allocation state. Provided there is only one Time Allocation Code for the equipment.


  • Form 1 is a dynamic form, the fields are determined by the LoaderQuery Module config.
  • Any autocomplete's available is sourced from this query.

Basic WorkFlow (Ready State)

  • Populate the Source field. 
  • Select an activity from the activity wheel (based on the value of the StartSessionReasonCode an activity might already be selected)
  • Based on the source loctation Code, the rest of the fields are autopopulated.
    • Cycling through the wheel, adds a new TimeAllocation activity.
      • This gets persisted to the StagingTimeAllocations table.


Other States

  • Selecting any of the other states prompts the user to specify the Time Allocation Code
  • Additional information can be requested from the user, using the query specified in the StagingStateWriteback module config from MiTime




Download Locations

  • To Download new Locations, go to the download page


  • The Page groups the locations, using  the LoaderLocationsQuery  module config.
  • Select the group header to download all locations under that group, or select individual location to download only the specific locations.


  • To search for a specific location, use the search bar to filter the list 


  • A ReadOnly list of the all downloaded locations are visible if the download icon is clicked
