Product: MineOperate Foundation : Workflow - Time

Created by Usuario desconocido (dask), last modified by Usuario desconocido (kins) on mar 17, 2020
  • Login workflow is similar to other apps.


  • Once logged in, select one of the Reason Codes

  • Select a location in the form (2).
    • The form is not configurable via SilverLight.
  • Toggle through each of the activities in the wheel, to log an activity.
    • Each activity corresponds to the current Reason Code (Dozer Push in this case), and the current selected equipment Type(DOZ816).
    • Cycling through the wheel records, 1 row in the stagingTimeAllocation table for the current Reason, and 1 row for each activity selected.


The Ready State signifies that the equipment is currently operating.

The other states are used to signify situations where the equipment is used to log reasons due to which the equipment is not operating.

The tabs are completely data-driven, and can be configured by adding more  Reason category and linking them to the reason codes for the current equipment type.



  • To log a non-operating time allocation, select the state and then one of the reason codes.
  • Complete the form, 
    • navigating away from the state saves it and ques it up for upload.

    • The form is derived from the writeback query specified in the StagingStateWriteback module config.
