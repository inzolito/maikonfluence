HxGN MineOperate Platform : On-board Packages standard

Created by TEODORO Eduardo on oct 22, 2018

Protobuf

Base Package for Equipment

For now on, all packages created by any module of MineOperate that came from a piece of equipment should conform to this standard. As time goes, old packages will be replaced.

Protobuf do not provide inheritance, so to have a similar feature we will use standardization of indexes for base fields.

In order to allow low level generic analysis and also off-line playback, the time and ownership of all packages will be done using this standard:

PropertyProtobuf indexRequiredDescription
eventDate1YesDate of the event of the package. Once the system is async and allow buffering, all packages should contains its own date/time in UTC
equipmentId2Yes

Unique Id of the piece of equipment (same Id of server database)

reserved3 to 5NoReserved to future extensions

How each package should start:

message BaseMessage
{
   required uint32 eventDate = 1;
   required uint32 equipmentID = 2;
   reserved 3 to 5;
}
