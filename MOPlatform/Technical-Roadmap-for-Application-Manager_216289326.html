HxGN MineOperate Platform : Technical Roadmap for Application Manager

Created by TEODORO Eduardo, last modified on mar 03, 2020

Introduction

Application manager is a back-end module developed in .Net framework 4.62 that offers agnostic application version management. It is service oriented and can provide binary transfer for applications versions in different ways.

The module can be separated in three different parts:

Binary storage

This part is agnostic of any other product. It uses Postgres to store data and can store binaries for multiple versions of each application.

The basic organization is: ApplicationType (Name, Platform) contains Applications (Name, Key) that contains ApplicationVersions (Version, Variant, Binary)

Version Assignment

This part depends on a list of equipment and devices, today it get these lists from UG Pro, it uses Equipment (Id, Name), Equipment Type (Id, Name), Device (Id, Name, Status), Device Model (Id, Name, Platform, Variant)

Desktop apps do not use this component, client apps just assume that the latest version on server must be installed for all users, which is not desirable for on-board applications.

Version Transmission

Current implementation offers three ways to transfer version binary: MQTT, REST and .Net Library. The client should start the workflow to transfer the binary.

To enable MQTT transfer it requires an MQTT broker, in that case, the version will be fragmented (configurable size) and is possible to follow up the download progress as well as cancel the transmission.

The MQTT transmission also implements a load balancer mechanism to prevent taking too much network and affecting operation data.

Once downloaded and installed, the client should inform the success of the installation.

Other transmission methods can be implemented, it can be configured by "Application".

Desktop applications also don't use this component, they just use standard protocols (WCF and REST) without any optimization to download and install new versions, which is fine for an office network.

Challenges

Current version is Windows only (.Net Framework)

The current client tool is a .Net Desktop tool

The module depends on UG Pro Equipment/Device list
