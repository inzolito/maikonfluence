1Map Rendering

FAILED

7 abr 2025GEORGE Nathan appears to be fixed in 4.0.3. OSA v1.2.9 & OPPP v4.1.2

Objects on lower portion of map are not being rendered properly, seems related with unintentional vertical offset when notification message showing

The map objects should be drawn properly inside visible map area, and not cropped
2Unattended Mode Activation

FAILED


Equipment Panel automatically restarted when Unattended Mode global configuration changes on server.

On test bench where we have Grader running shell-app/oppro-plugin, it is also being restarted

Equipment Panel automatically restarted when Unattended Mode global configuration changes on server.

Equipment panel affected is only Truck and Shovel.

3Unattended Mode Activation

FAILED


After unattended mode configuration changed on server, the notification to operator is showing as ERROR CODE which may mislead the operator

After unattended mode configuration changed on server, the notification to operator treated as an Informational types of message not to confuse it with actual error.

4

Shovel/Truck - Selecting Assignment Type

FAILED

 

First time opening the Unattended Mode view after onboard app started, the list selection is empty

Upon opening the view, the default list should be automatically populated and visible for user to choose
5Shovel/Truck - Selecting Assignment Type

FAILED

 

The visible list is always from the top of the list by default.

(i.e: Operator won't be able to know what is the current Blast from Panel directly)

When the list is opened, the currently active item (i.e: Current Blast Location / Lock) should be pre-selected and visible for operator

6Truck change grade

FAILED

 

There is an incomplete informational message on Shovel when changing grade (grade and material information is missing)

How to reproduce:

  1. Truck is loading to Shovel
  2. Shovel operator change to different Grade by clicking on Material widget and then SET GRADE, or from selecting grade layer and then SET GRADE
  3. When Shovel is finished Loading and goes to Waiting, the message will be shown.


7Truck/Shovel Clearing Assignment / Lock

FAILED

 

SYSTEM-DEFINED is showing as CLEAR

The item selection/message should be referring as SYSTEM-DEFINED as per UI Specs.
8Truck UI Panel (Tire & Breaks)

FAILED

 

There was a change made in 4.0-0001: https://jira.hexagonmining.com/browse/MOPS-27139, in which the code will no longer read the dock_items configuration.

This has an undesired affect on Truck Panel, where there is no way to hide oppro-tire dock item on Truck panel

The oppro-tire dock items (Tire & Breaks feature) should only be visible to applicable equipment, as it is dependent on customer having specific OEM interface available.
9UI Panel (Diagnostics Menu)

FAILED

 

When Diagnostic Menu screen is opened, the main dock bar is gone, the title bar suddenly positioned under the view.

See below:

Diagnostic Menu screen should be displayed consistently as in other view when opened, and not interrupting the main dock bar and title bar position
10UI Panel (Engine Hours)

FAILED


Engine Hours value's vertical position is not inline with the label


This is observed especially when we don't have the configuration 'disable_hourmeter_sensor', or it was set to false

disable_hourmeter_sensor=false
 
11UI Panel (Engine Hours)

FAILED


It is not possible to put engine hours value manually, maybe because the cursor is not visible, so always ended up failed the validation.

It was observed with the following configuration:

validate_hourmeter_with_range=true
disable_hourmeter_sensor=true


Operator can enter Engine Hours value manually when configuration put in place.
12UI Panel (Prestart)

FAILED


When operator typing prestart comment, the font size is unproportionally really small even though the text area is quite large.

Prestart Comment should be readable and have proportional size
13UI Panel (Refuel view)

FAILED


On the Refuel view, it seems the Apply button is always enabled irrespective of values entered in the Fuel Amount and Engine Hours fields.


According to figma mockup, the Apply button only gets enabled when we enter the values in both the fields.

14Unattended Mode for Truck: Single Assignment

FAILED


Raised from Tech Pub team during the validation on bench, we only have the Single Assignment to Shovel, which only showing list of Shovel:

The UI Specs for Unattended Mode for Truck has the Single Assignment menu, in which the list is dynamic based on current truck activity (either loaded or not loaded):

Refer to OP Pro Unattended Mode UI/UX Design, on the topic 4. Creating Single Assignment .. page 6-7

15Unattended Mode Activation

FAILED


When unattended mode changed (i.e: from disable to enable, or vv), any status change from equipment that require dispatcher approval (auto_accept=false) should get overrides (no longer showing in Dispatcher's Inbox) only if JAMS on equipment is restarted manually
