Product: MineOperate OP Pro : Application Architectures supported in 3.x

Created by GEORGE Nathan, last modified on nov 07, 2022

The OP Pro 3.0 release supports a hybrid 2.8 and 3.0/3.1 Application Architecture


2.8 Architecture
  • Legacy Ruby GTK LP and Ruby GTK + OreNav HP 32 bit applications supported
  • Applications and components installed and run on the Display 9
  • JAMS functions as the application service between the Onboard device and Server
  • All Server to Field device communications and data replication is done through JAMS
  • Core & UHP devices are used as a sensor & communications device

3.0 Architecture
  • Web application runs in Chromium browser on Display
  • Application Services & Database (GVOS, JAMS, SQLite) run on the Core device
  • GVOS functions as the application service between the Onboard device and Server
  • All Server to Field device communications and data replication is done through GVOS
  • The use of the Onboard web applications can be managed through device configurations.  This allow customers to maintain the use of 2.8 applications but migrate to 3.0 onboard web applications in a managed way.
  • UHP is not supported to run the 3.0 Architecture due to it not meeting the minimum CPU processing requirements.
  • Currently, only Shovel LP and Truck LP web panels are supported.  However, Aux LP and HP web panels will be supported in future OP Pro releases.
