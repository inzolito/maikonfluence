Product: MineOperate OP Pro : JView

Created by GEORGE Nathan, last modified on nov 15, 2020

The Jigsaw Jview business intelligence (BI) standard report suite was developed to provide out-of-the-box reporting functionality at installation and seamless upgrades and enhancements for reporting on the Jigsaw Jmineops system.  There are many components that encapsulate BI in Jview.

Information provided by Jmineops enables users to have greater control of the operations and production environment.  Jmineops optimizes and centralizes equipment tracking, dispatching and diagnostics ensuring that activities and operators can be directed, material movement can be confirmed, and machine health can be monitored within a comprehensive pragmatic reporting structure.  

The product suite is built on Microsoft SQL Server 2008 R2/2014 Standard (MSQL) technologies.  It follows industry standard methods for data warehousing and report structure.  This includes SQL Server, Reporting Services (SSRS) and Analysis Services (SSAS).  The Jview web application serves the reporting components to the end user utilizing the Report Server as content.

Since there are many levels of data consumers at a mine (even within a mining company, having many separate mine sites) Jview is structured to provide content tailored for different use and analysis.  


The standard report suite comprises a combination of content depending on the need for information, the interval, quantity, and purpose.

  •  Dashboards for near real-time information.  Generally designed to provide operational intelligence focused on change, and proactive action.  The objective is having the most important information, a consolidation and arrangement of data in one view to achieve goals or identify problems.
  •  Static layout reports for common daily and operational reports.  Generally designed to be printed and or emailed, for shift, daily, and weekly on production, maintenance, and operational metrics.  Some reports are laid out like scorecards for general month to date and aggregated reporting.
  •  Dynamic layout pseudo pivot reports for common longer term analysis, allow for some flexibility in aggregated exploration but still within a standard web browser.  They are used to access the SSAS database cubes through Jview without needing Excel, for greater periods of time.
  •  SSAS OLAP database cubes for ad-hoc reporting and in depth analysis.  Modelled for slicing and dicing data over greater dimensions and periods of time with pivot charts and pivot tables.  Cubes are browsed using Microsoft Excel™ 2007 and greater with a standard OLE DB connection.  Workbooks can be saved locally, emailed or shared within the common Microsoft framework of tools and applications.  There are excellent analytics add-ins for Excel including PowerPivot (2010), PowerView (2013), and PowerMap (2013) that make data interaction rich and insightful.

Jview renders dashboards, static and dynamic reports in the Jview web application that can be viewed on network computers and many other devices with internet browsers connected to the WIFI network, including mobile phones and tablets.  Please see the Jigsaw Jview Standard report list for more detailed explanation.

Custom extended content utilizing any of the components of Jview can be created, appended or added.  The platform allows for modifications and extension to existing standard content or new reports, dashboards, or cubes.  Clients can own and customize their data and the artifacts.  Content can be shared between installations and sites as well.  Creating and extending content is done easily with Microsoft SQL Server Report Builder™ 3.0.

Jview is a simple, flexible platform that is extendable and allows customization to content delivery across the operation.  It is critical to get supervisors the information so they can make decisions on the spot, out in the field.  With many new technologies; smart phones, tablets and 3G networks, data can now be pushed out more efficiently to these devices.  Interface layout and design is critical because visualizations are viewed differently on various devices and situations.  Visual analysis contributes significantly to the discovery, interpretation, and insight.   Matching content with users’ skills and purpose enhances interactivity and engagement.

From dispatcher, to analyst, to management, Jigsaw makes it possible for individuals and teams to control the environment and discover the information they need while subtly hiding the fact that they are using data and instead gaining knowledge, when it has to be right.

2. BI - JSON Data Extension Installation & User Guide

Overview

Some of the reports developed by Jigsaw require a custom data source that was written to provide JSON data directly from JAMS objects. This is a C# library that converts the JSON data to a form that can be used by SSRS.
This page gives instructions on how to install and use the extension.

JAMS Web Service Setup for Nginx (Linux Server)

Nginx Configuration

Nginx is a free, open-source, high-performance HTTP server that is used to power the web service for the JSBI JSON data extension to use. It must be configured to service the requests for object data. 
If you've installed nginx from the server-i386.tar.bz2 or server-x86_64.tar.bz2 packages, nginx will already be installed in /opt/local/sbin.
Perform the following steps for nginx setup on a new server:

  • $ sudo mkdir -p /etc/nginx
  • $ sudo mkdir -p /var/log/nginx
  • $ sudo mkdir -p /var/lib/nginx/body
  • $ cd /etc/
  • $ sudo tar xf nginx_cfg.bz2 (found at bpv)

Then add the line "user www-data www-data;" to the top of the nginx.conf in /etc/nginx if it isn't already there. 
By default the /opt/Jigsaw/Sources/Documentation/html directory is shared in the nginx.conf 
You may want to change this to something you can test with if you don't already have this directory. 
For example, change the line root /opt/Jigsaw/Sources/Documentation/html ; to root /opt/Jigsaw/Clients/client/html;

  •  $ sudo /opt/Jigsaw/Services/nginx start

This will test the basic setup of nginx. You can then create a symlink to the Services script in /etc/init.d/ and register it.

  •  $ cd /etc/init.d
  • $ sudo ln -s /opt/Jigsaw/Services/nginx nginx
  • $ sudo chmod +x /etc/init.d/nginx
  • $ sudo /usr/sbin/update-rc.d -f nginx defaults
  • You can then start nginx like you would any standard init.d service.
  • $ sudo /etc/init.d/nginx start

 

JAMS Configuration


The JAMS configuration file xxx.jams must have entries for the business intelligence bundle and the fastCGI module. The following entries are needed:

 [bundle JBusinessIntelligence]
[fastCGI port=localhost:5000]
The port number can be any unused port, but must be the same as what is setup in the /etc/nginx/nginx.conf file for the fastcgi_pass parameter. (In the entry above port 5000 is used)

 Nginx/JSON testing

If the text based browser lynx is installed (sudo aptitude install lynx) or you can connect from the network over 8080 to the JAMS server, try the following to test if nginx is now running:

  • $ lynx localhost:8080

This should show either your index page, or a 403 forbidden if you have not set one up.
To then test the JSON interface try:

  • $ lynx localhost:8080/jams/objects/Device

or

  • $ lynx localhost:8080/jams/objects/Equipment

This should then prompt you to download since browsers do not normally recognize the file type. The downloaded file should look similar to this:
[{"mac_address":null,"device_type":null,"utilities":null,"id":1,"software_version":null,"name":"APPLICATIONS","groups":null,"netmask":null,"address":null,"updated_at":"2011-03-08 08:58:09","installed_version":null},...


Library Installation


Downloadthe DLL zip file from the server location:
Go to http://hydra/Releases/TotalView2 and download the <release>/JSBIJAMSDataSource_<release>.zip from the appropriate release.
Right click and extract in the same folder and copy the DLL to C:\Program Files\Microsoft SQL Server\MSRS12.MSSQLSERVER\Reporting Services\ReportServer\bin. 

If this is an update to the DLL then simply overwrite.

IMP NOTE: It is a good idea to make the backup of the config files before you modify them (rsreportserver.config,rssrvpolicy.config)
Add this to the data section of the C:\Program Files\Microsoft SQL Server\MSRS12.MSSQLSERVER\Reporting Services\ReportServer\rsreportserver.config:

<Extension Name="JAMS" Type="Microsoft.Samples.ReportingServices.JAMSDataExtension.JAMSConnection,Microsoft.Samples.ReportingServices.JAMSDataExtension"/>

 Add this entry in the C:\Program Files\Microsoft SQL Server\MSRS12.MSSQLSERVER\Reporting Services\ReportServer\rssrvpolicy.config. Use this entry as the last group in CodeGroup:

<CodeGroup class="UnionCodeGroup"

      version="1"

      PermissionSetName="FullTrust"
      Name="JAMSCodeGroup"

      Description="Code group for my JAMS data processing extension">

      <IMembershipCondition

              class="UrlMembershipCondition"

              version="1"

              Url="C:\Program Files\Microsoft SQL Server\MSRS12.MSSQLSERVER\Reporting Services\ReportServer\bin\Microsoft.Samples.ReportingServices.JAMSDataExtension.dll"

       />

</CodeGroup>

NOTE: Please verify the Url to the DLL is correct by copy pasting the above path into explorer. On some systems the SQL server reporting services may be installed on a drive other than the C: drive. Also on 64 bit installations with a 32 bit version of the report server would be in "Program Files x(86)" instead of "Program Files". Alter the URL if necessary to the appropriate path.

Then change this part of the same file (rssrvpolicy.config):

                         <CodeGroup 

                            class="FirstMatchCodeGroup"

                            version="1"

                            PermissionSetName="Nothing">

                        <IMembershipCondition 

                                class="AllMembershipCondition"

                                version="1"

                        />

by substituting PermissionSetName="Nothing" with PermissionSetName="FullTrust"


3. BI - JView and Administration Utility Installation  

Overview


These are the instructions for installing or updating JView and the JSBI administration tool.

Installation on the Target Server


This section will describe how to install the web application on the target site.

 Pre-installation for First Time Install


The JView web application requires some additional software that is not usually installed on new target servers. Follow these steps to prepare the server to run the installer web application.

  • From Hydra http://Hydra download the WebDeploy Tool - x64 found under Jview (TotalView 2) > Extra Jview Related Windows Tool Installers section. 
  • As an administrator, run the installer (select Typical if asked during installation) to add web deploy to the server.
  • IMP NOTE: Report Control Installer IS NOT REQUIRED for SQL Server 2014 (x64) on Windows Server 2012 R2 (x64).

 

Disabling the Website

If this is an update to JView perform the following steps:

  •  Access the IIS7 management panel.
  • Navigate to the Default web site which is where the JView web application lives.
  • Right click on the node and navigate to the Manage menu and click on Stop. The website is now not available.


Installing the jView Web Application

  • From Hydra http://Hydra download the <release>/JSBIViewerInstaller_<release>.zip file.

  • Extract all from the downloaded zip file. You can normally do this in the Documents directory of the account you are logged in with.

  • Open the extracted folder then open the JSBIViewerParameters.SetParameters.xml file using notepad.

  • Change the connection strings to the appropriate values for the installer command to use and save.

  • Open a command window with the "Run as Administrator" option and change directories to the expanded installer files directory created above.

  • Execute the command JSBIViewerPackage.deploy /t which executes the installer with a "what if" scenario. This will verify your database connection strings are valid.

    If you get a connect error then go back and correct the mistakes in the connection strings and run the command again.

  • After you verify the website deployment can finish without error run the command JSBIViewerPackage.deploy /y which will perform the installation (or update if the website was previously installed.)

  • For first installations bring up the website and verify it's operational (Note at this point no BI Content has been added).

  • If this is an update, bring the site back up with the following steps:

    1. Access the IIS7 management panel

    2. Navigate to the Default web site which is where the Jview web application lives

    3. Right click on the node and navigate to the Manage menu and click on start

    4. The website is now available (Note that current BI Content has not yet been upgraded at this point)

Installing the JSBI Admin Tool


The JSBI administration tool allows update/installation of the business intelligence artifacts that drive the JView web application. To install the tool perform the following steps.

  • From Hydra http://Hydra download the <release>/JSBIAdminToolInstaller_<release>.exe file.
  • If this is an update use the windows control panel to remove the current JSBIAdmin tool.
  • Change directories to the expanded installer files and execute the .exe file.
  • Follow the directions to install the application.

 

4. BI - JView Website Artifact Installation  

Overview

These are the instructions for installing or updating JView report content and supporting artifacts using the JSBIAdmin tool.
This tool updates standard artifacts including: Microsoft SQL Server database scripts (.sql files), Microsoft Analysis Server cube definitions and other XMLA files (.xmla files),
Report Shared Data Sources (.rds files), Report Shared Data Sets (.rsd files), Reports (.rdl files) and Excel Analysis Spreadsheets (.xsls files)


Report Artifact Installation Steps

  •  From Hydra http://Hydra download the <release>/JMineOPS_<release>.zip file of the Jview (TotalView 2) section
  • Login as a local administrator (e.g., JSBIAdmin user), to the Windows server and copy the zip file to a place where you are allowed to copy files,
    usually the desktop is a good place for this file
  • Right click on PowerShell and run as Administrator
  • Execute the following command: set-executionPolicy unrestricted , then press enter
  •  Type "Y" to confirm, then press enter
  • Change directories to the location where your installer is located:
    cd C:\Program Files <x86>\HexagonMining\JSBIAdmin
  • With the use of the JSBIAdmin installer deploy the artifacts to the ReportServer as shown in the following example; in fact this can be copied and edited for your use:
    .\JSBIAdmin.exe -As localhost -Au JSBIAdmin -Ap Jigsaw1 -Ad ReportServer     -A <Directory Path>JMineOPS_<release>.zip -Wu JSBIAdmin -Wp Jigsaw1 -W http://localhost/ReportServer/ -S localhost -Ju JSBIAdmin -Jp Jigsaw1 -Js localhost -Jd jmineops -JSONURL http://<FloatingIP>:8080/jams/objects/ -TVWebApplicationRoot C:\inetpub\wwwroot\JView
  • Repeat these steps as necessary (i.e., For as many bundles as required) , Note that due to some dependencies the JMineOPS Standard bundle must always be deployed before any other bundles
     

JSBIAdmin installer argument description:


  • [-S  Analysis server host name (defaults to localhost)]
  • [-d Analysis server database  (defaults to jmineops)]
  • [-As SQL server host name (defaults to localhost)]
  • [-Au SQL Server user name (defaults to JSBIAdmin)]
  • -Ap SQL server password
  • [-Ad Database for the report server on the SQL server (defaults to ReportServer)]
  • [-A  Input Artifact ZIP file.  Defaults to C:\Users\JSBIAdmin\Documents\BI_JMineOps_1.057.zip ]
  • [-W  Report server web service URL.  Defaults to http://localhost/ReportServer/]
  • [-Wu Report server web service username. defaults to JSBIAdmin]
  • -Wp Report server web service password
  • [-Js Jmineops SQL server.  Defaults to localhost.]
  • [-Ju Jmineops SQl server username. defaults to JSBIAdmin]
  • [-Jd Jminops datbase on the SQL server.  Defaults to jmineops.]
  • -JSONURL  URL to the JSON data service that serves JAMS data.
  • [-TVWebApplicationRoot path to the Totalview web application.  Defaults to C:\inetpub\wwwroot\JView]
