OP Pro Manager

OP Pro Manager only accessible within Azure internal virtual network (10.0.1.x)

To run OP Pro Manager, RDP to the OP Pro Reporting server and run OP Pro Manager from the desktop

  1. Press the Windows key.
  2. Type Remote Desktop Connection
  3. Use the following connection and login 
    • Computer: 52.148.176.140
    • Username: opprodemo
    • Password: XX OP Pro - Passwords.docx
  4. Once connected to the server, open OP Pro Manager from the desktop shortcut using
    • Server: mineops / 10.0.1.4
  5. To connect using Dispatcher role (will allow you to receive messages and exceptions from the Web Panel), provide the following access:
    • User: dispatch
    • Password: X OP Pro - Passwords.docx
  6. To connect using Admin role (full access to views, but will not receive exceptions in inbox), provide the following access:
    • User: admin
    • Password: XXX OP Pro - Passwords.docx

VNC to equipment

Remotely connecting to equipment display should work as it is using VPN address of equipment.

  1. From OP Pro Manager menus, click Operations > Equipment Operations view (or other specific equipment type view such as Trucks, Excavators, etc)
  2. Choose the equipment from the list (use test bench below when available)
  3. Click on VNC

Note: A local host entry for the router role needs to be created to support the MQTT service

  1. Press the Windows key.
  2. Type Notepad in the search field.
  3. In the search results, right-click Notepad and select Run as administrator.
  4. From Notepad, open the following file: c:\Windows\System32\Drivers\etc\hosts.
  5. Add the following entry. 13.77.174.169 router
    or 40.125.64.191 router (if you wish to use the Shovel simulator)
  6. Select File > Save to save your changes.
