Onboard PanelTruck PanelLoading2.05 and aboveAuto Loading on DepartureWhen a truck does not have a payload sensor, prevents the activity from going from spotting to loading automatically and requires the operator to set loading manually
moduleattributevalueapplication
actionauto_loading_on_departure<true/false>JAMS

Impact
Today the behavior of the system is to automatically change from spotting to loading, and this will continue to happen when the auto_loading_on_departure configuration is set to value True or is not configured.
When auto_loading_on_departure is set to False, the system will not change automatically from spotting to loading. This new behavior (value = false) is a required behavior when the mine does not have payload sensor and the operators are instructed to set the "loading" activity manually. This will prevent the system generating inconsistent records in the database.

Onboard PanelTruckFueling2.8 and aboveForce fuel input after login

Operators will be prompted to enter fuel amount after refueling when Operator logs back into equipment.  The fuel prompt will only appear if the equipment is set to the refueling reason code.

Note: if the force_manual_fuel_input configuration is "true", fuel input will be requested only once for each fueling process. If it's "false" (or the configuration doesn't exist), the fuel input will be requested at each login as long as the truck is in fueling reason.
