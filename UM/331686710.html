Initiate the Leap Quick to Market development 

GEORGE Nathan GEORGE Nathan 

This development is an outcome of completing the LEAP Quick to Market Assessment objective. The result of that assessment have shed lights on what should be the next steps in order to obtain an OP Pro version able to be run on MASEP OS in a standalone scenario.

Hence, it was decided to deprioritize the development related to the Port of OP Pro to MASEP SDK in favor of the changes needed to make OP Pro ready to run on MASEP OS as per the Leap Quick to Market scope.

Replace JWS with the new GVOS API Wrapper

GEORGE Nathan 

GEORGE Nathan 

It was identified on flight during PI#2 that a new solution targeting to deliver the same capabilities provided by the Jigsaw Web Service (JWS) is available on the GVOS API via an API Wrapper.

Hence, and in addition that, considering the challenges that were been faced with the adjustments to use the JWS with OP Pro version 4.0-002, it was decided to focus on make OP Pro 4.0-002 release with the GVOS API Wrapper in replacement to the JWS taking into consideration that the endpoints provided by the Wrapper meets the customer needs regarding the reporting services and put os back on the way to deprecate the JWS.
